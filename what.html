What is MOAP?

MOAP stands for the Mother of All Programs

How does it work?

The Mother of All Programs (MOAP) was started in the Fall of 2015 with the intention of developing a program that would automatically design computer algorithms.

The project was a success in its infant stages, solving a variety of simpler algorithm problems and demonstrating that automated algorithm design is a very real possibility.

This is fortunate, because until now, algorithm design has always been the realm of a few very highly-trained, highly-paid individuals in computer science, who are hard to find and even harder to hire.

What is an example?

They would first rewrite and formalize the problem into our input syntax:

"Given list a, list b, list c; Find d st child(a,d) and child(b,d) and !child(c,d)"

The program will then output the pseudocode to solve the problem, the most recent example of which is included here:

na = new HashSet(c) nb = new HashSet(b) foreach child d of a if !na.contains(d) if nb.contains(d) return [d]
